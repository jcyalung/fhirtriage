# FHIRtriage
A stateful multi-agent system built with Next.js and LangGraph that leverages clinical NLP to map natural language symptoms to ICD-10 codes and utilize a LightGBM classifier to determine safe triage decisions.

## Disclaimer 
This is a proof-of-concept project for educational purposes. The outputs, recommendations, and routing decisions generated by the ML models and AI agents in this project are for demonstration purposes ONLY. It is not intended, designed, or cleared for clinical use, medical decision-making, or live patient triage. 

To ensure absolute privacy, all patient profiles, clinical histories, and Electronic Health Record (EHR) data present in this repository/application are 100% synthetically generated. No Protected Health Information (PHI) is used, ensuring zero risk of HIPAA violations or privacy breaches. Any resemblance to real persons, living or deceased, is purely coincidental.

## Tech Stack
### Frontend
- Next.js with App Router
- TailwindCSS 
- Server-Sent Events (SSE) native web API
### Backend
- FastAPI (Py3.11)
- LangGraph v1.2
- Pydantic v2
- `fhripy`
### ML & AI
- XGBoost (with `scikit-learn`)
- `shap`
- Llama 3.1 (8B), or DeepSeek-R1
### Infra and Database
- Docker
- HAPI FHIR JPA
- Synthea
- PostgreSQL with `pgvector`
- Ollama
